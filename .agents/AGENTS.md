# Style Rules & Guidelines

- **Color Palette & Theme Customizations**:
  Always use the CSS custom properties defined in [global.css](file:///e:/v1/src/styles/global.css) for all UI styling. Do not introduce arbitrary colors or hardcoded hex/RGB values.
  - Backgrounds: Use `bg-[var(--card)]` or `bg-[var(--background)]`.
  - Borders: Use `border-border` (which uses `var(--border)`).
  - Text: Use `text-foreground` (for `var(--foreground)`) and `text-muted-foreground` (for `var(--muted-foreground)`).
  - Accents & Highlights: Use `var(--accent-color)` (e.g., `text-(--accent-color)`, `bg-(--accent-color)`, or `border-(--accent-color)`).

- **Blog Writing Style**:
  - Never use em dashes (`—`) anywhere in blog post content or frontmatter. Rewrite the sentence or use a comma instead.

- **Blog Post Header Images**:
  When generating a header image for a new blog post, strictly follow these rules:
  - **Reference image**: Always read `public/assets/blog/father-forgets.png` first to understand the exact visual style before generating.
  - **Style**: Minimal hand-drawn line-art sketch. Cream/off-white background (`#faf6ef`). Thin dark single-stroke lines only. No color fills. No shading. No gradients. Stick figures are preferred.
  - **Dimensions**: Exactly `1024 x 576 px` (16:9). If the generated image is a different size, center-crop it using Python PIL to match these exact dimensions before saving.
  - **Format**: PNG, RGBA mode.
  - **Destination**: Save to `public/assets/blog/<slug>.png` where `<slug>` matches the blog post filename.
  - **Caption**: Always add an italicized centered caption below the image in the markdown: `Illustration generated by ChatGPT`.
  - **Alt text**: Use a descriptive alt text that reflects the content of the image, not a generic label.
